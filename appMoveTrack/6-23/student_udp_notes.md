## 程序大概功能

appMoveTrack 是一个轨迹可视化和回放软件。

主要功能：

- 加载轨迹数据文件，例如 txt 或 csv。
- 按对象 ID 拆分多条轨迹。
- 在地图上显示轨迹。
- 支持回放轨迹，显示飞机或目标的运动过程。
- 支持 3D 视图。
- 支持 UDP 主从通信，主机把回放帧发送给从机。
- 支持转发 TRACK 格式的数据包到指定 UDP 地址和端口。

## 关键端口

- Qt 软件主机端口：8899
- 从机默认接收端口：8900
- TRACK 转发监听端口：9001
- 软件本地转发发送端口：9101

当前转发配置：

- 目标端口：9001
- 本地发送端口：9101
- 通道已启用

## 已经做过的排查

### 1. Qt/C++ 测试端自检

命令：

```powershell
..\bin\udp_test_endpoint_qt.exe selftest
```

结果：

```text
SELFTEST PASS
```

说明：

Qt/C++ UDP 测试端本身没有问题，本机 UDP 收发能力正常。

### 2. 监听 TRACK 转发

命令：

```powershell
..\bin\udp_test_endpoint_qt.exe listen --bind 127.0.0.1 --port 9001 --duration 120 --require-track
```

作用：

监听软件是否往 127.0.0.1:9001 发出 `TRACK,...` 数据。

如果结果是：

```text
总包 0，TRACK 包 0
```

说明：

软件当前没有往这个端口发 TRACK 数据。常见原因是软件没有进入实际发送状态，例如没有启动主机、没有真实接收链路、没有播放数据，或转发通道没有生效。

### 3. Qt/C++ 模拟从机

命令：

```powershell
..\bin\udp_test_endpoint_qt.exe slave --host 127.0.0.1 --host-port 8899 --client-port 8900 --ready --duration 120
```

作用：

让 Qt/C++ 测试端假装成一个从机，连接 Qt 软件主机。

正常应该看到：

```text
ACCEPT
SET_READY
FRAME_BEGIN
FRAME_TRACK
FRAME_END
```

如果出现：

```text
[WinError 10054] 远程主机强迫关闭了一个现有的连接。
```

当前解释：

这通常不是测试端命令写错，而是 127.0.0.1:8899 没有软件主机在监听。也就是软件虽然打开了，但还没有点击启动主机或主从一体。

## 已经修过的问题

### 1. 重复软件进程

发现电脑里同时运行了两个 `appMoveTrack` 进程。

处理：

已关闭重复进程，避免 UDP 状态混乱。

### 2. 转发端口冲突

原来配置里：

- 本地发送端口：9001
- 目标接收端口：9001

测试端也在监听 9001，可能导致软件发送端和测试端接收端抢同一个端口。

处理：

已改成：

- 本地发送端口：9101
- 目标接收端口：9001

### 3. 测试脚本错误提示不清楚

原来 Python 辅助脚本模拟从机连接不上主机时，会刷很多 `WinError 10054`。

处理：

已改成更容易理解的提示：

```text
未连上 Qt 主机：请先在软件里启动主机/主从一体，并确认 8899 端口正在监听
```

## 下一步操作顺序

### 方案 A：验证软件主机到 Qt/C++ 从机

1. 打开 Qt 软件。
2. 在软件里进入节点配置。
3. 选择主机模式或主从一体。
4. 点击启动主机。
5. 在 PowerShell 运行：

```powershell
..\bin\udp_test_endpoint_qt.exe slave --host 127.0.0.1 --host-port 8899 --client-port 8900 --ready --duration 120
```

6. 软件里加载数据。
7. 软件里点击回放。
8. Qt/C++ 测试端应该收到 `FRAME_TRACK`。

### 方案 B：验证软件 TRACK 转发到 Qt/C++ 监听端

1. 先运行监听：

```powershell
..\bin\udp_test_endpoint_qt.exe listen --bind 127.0.0.1 --port 9001 --duration 120 --require-track
```

2. 软件里确认转发通道启用。
3. 软件进入主从一体或有真实从机接收的状态。
4. 软件加载数据并点击播放。
5. Qt/C++ 测试端应该收到 `TRACK,...`。

## 回环验证说明

这一步可以叫：

- UDP 本机回环验证
- UDP 主从通信联调
- Qt/C++ 模拟从机接入测试
- Qt 软件 UDP 主机发送链路验证

更完整的说法：

我用 Qt 6.8.3/C++ 测试端模拟从机，向 Qt 软件主机端发送 JOIN 请求，验证软件能否接收从机连接、返回 ACCEPT，并在播放轨迹时发送 FRAME_TRACK 轨迹帧。
